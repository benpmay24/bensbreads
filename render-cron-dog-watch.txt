# Render Blueprint — Dog Watch collector runs as a Cron Job, separate from the web service.
#
# In the Render dashboard, create a Cron Job with:
#   Schedule:  0 6 * * *          (daily at 6:00 AM UTC)
#   Command:   python manage.py dog_watch_sync
#
# Each run is fully self-contained:
#   1. Downloads the USDA active licensees spreadsheet (adds new sites, updates changes)
#   2. Checks APHIS for new inspection reports on each facility due for refresh
#      (already-stored report URLs are skipped; violations_parsed=True rows are never re-parsed)
#   3. Parses all pending report PDFs into violation records, updates addresses and counts
#
# Already-processed reports (violations_parsed=True) are never touched.
#
# First-time backfill (run once via Render Shell if the database is empty):
#   python manage.py dog_watch_sync --force
#
# If a run gets stuck:
#   python manage.py dog_watch_sync --clear-lock
#
# Clear lock and immediately start a new run:
#   python manage.py dog_watch_sync --clear-lock --force
#
# Optional: parse violations only (no USDA import or APHIS checks):
#   python manage.py dog_watch_sync --parse-violations-only
#
# Optional env vars:
#   DOG_WATCH_VIOLATIONS_BATCH_SIZE=40   (reports per parse batch, default 40)
#   DOG_WATCH_VIOLATIONS_MAX_BATCHES=50  (max parse batches per run, default 50)
